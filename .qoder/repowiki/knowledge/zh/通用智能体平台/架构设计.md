- **分层架构**：采用经典的 Controller-Service-Config 结构。`controller` 层（如 `AgUiController`）处理 AG-UI 协议请求；`agent` 层（`AgentSessionService`）封装核心业务逻辑；`config` 层负责基础设施组装。
- **核心集成**：通过 `AgentConfig` 实例化 `HarnessAgent`，集成 `RedisDistributedStore` 实现分布式状态管理，并配置 `DockerFilesystemSpec` 提供 Session 级沙箱隔离。
- **多租户机制**：利用 `TenantContextHolder` WebFilter 从 HTTP Header 提取租户信息并注入 Reactor Context，在 `AgentSessionService` 中构建复合 `RuntimeContext` 实现数据隔离。
- **可观测性**：通过 `TracingConfig` 初始化 OTEL SDK，并由 `OtelTracingMiddleware` 拦截 Agent 执行过程，将 Trace 导出至 Langfuse 或 OTLP Collector。
